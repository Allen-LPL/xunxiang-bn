# 礼品卡（兑换码）功能增强 2026-07-06

## 需求
后台批量生成兑换码，支持兑换不同数额积分、不同数量商品；兑换码有使用与兑换记录；兑换并发控制。

## 现状与新增
- 批量生成：已有（礼品卡列表 → 生成卡密，单批上限 10000，按批次管理并可导出 Excel/二维码）。
- 不同数额积分：已有（data_type=2，卡上的 secret_value 即积分数；不同数额建多张卡分别生成）。本次加正整数校验。
- 记录：已有（plugins_giftcard_card_secret_log 尝试日志 + is_exchange/exchange_time 兑换记录 + use_data 实物领取记录）。

## 本次改动
1. 并发控制（核心修复）`CardSecretService::CardSecretExchange`
   - 原实现"先查后改"，并发下同一卡密可被重复兑换（积分/余额双花）。
   - 现改为事务内原子抢占：`UPDATE ... WHERE id=? AND is_exchange=0`，受影响行数<1 判已使用；发放失败回滚自动释放。
   - 新增防刷：同一用户 60 秒内失败 ≥5 次临时拒绝（防暴力遍历卡密）。
   - 新增作废控制：卡主体 is_enable=0（后台停用）即整批不可兑换。
2. 商品数量支持（data_type=3）
   - secret_value 支持「商品id:数量」格式，逗号分隔，如 `12:2,35`（商品12可兑2件、商品35可兑1件），兼容旧数据纯 id。
   - 下单抵扣（Hook::BuyGroupGoodsHandle）按件数跨卡累计抵扣；部分使用后卡密剩余件数可继续兑换（use_status 按件数判定）。
   - use_data 记录含件数（stock），订单取消/关闭按订单整体释放。
   - 兑换商品清单接口（exchangegoods）返回 exchange_num（剩余可兑件数）。
3. 加固
   - install.sql：secret_key 改唯一索引；生成时先写批次内唯一占位 key 再更新为正式卡密。
   - 存量库执行 upgrade.sql（执行前按注释先检查重复）。

## 上线步骤
1. 代码发布后执行 upgrade.sql（替换 {PREFIX} 为实际表前缀，如 sxo_）。
2. 后台入口：应用中心 → 礼品卡 → 新建卡（选类型/面额）→ 生成卡密（填数量）→ 批次下载发放。
